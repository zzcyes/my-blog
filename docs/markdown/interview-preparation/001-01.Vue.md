## 思考

阅读源码钱先思考一个问题：为什么我们要去阅读 `Vue` 源码?

分析如下代码：

```vue
<template>
  <div>{{ msg }}</div>
</template>

<script>
export default {
  name: 'home',
  data() {
    return {
      msg: '123',
    };
  },
  created() {
    console.log('before:msg', this.msg); // 123
    this.msg = '456';
    console.log('after:msg', this.msg); // 456
  },
};
</script>
```

是不是感觉很容易理解呢，都是`vue`最基础的常规写法。

接下来给自己提几个问题：

1. 为什么`data`是个函数

2. 为什么给 `data` 函数返回对象的`msg`赋值，`this.msg`却能够引用?

3. 给`this.msg`赋值，`data`函数返回对象的`msg`是否改变呢？若变了，那是为什么呢？

提这几个问题的目的，是让我们平常要多去思考，不要对很多东西都是理所当然的，要多去思考，了解其本质。有时候并非是“所见即所得”。

bad：

1. 一般都是用函数啊，`vue`本来就是这样的啊

2. 可以引用啊，`vue`本来就是这样的啊

3. 会变啊，`vue`本来就是这样的啊

good:

1. `vue`内部实现是支持`data`有两种类型的，分别为对象和函数，官方推荐使用函数。

> 当一个组件被定义，data 必须声明为返回一个初始数据对象的函数，因为组件可能被用来创建多个实例。如果 data 仍然是一个纯粹的对象，则所有的实例将共享引用同一个数据对象！通过提供 data 函数，每次创建一个新实例后，我们能够调用 data 函数，从而返回初始数据的一个全新副本数据对象。

2. 能过引用，`vue`内部是通过代理的方式去实现的。

> 可以通过 vm.$data访问原始数据对象。Vue实例也代理了data对象上所有的property,因此访问vm.msg等价于访问vm.$data.msg。

3. 会变，`vue`内部是通过代理的方式去实现的。

> Vue 将会递归将 data 的 property 转换为 getter/setter，从而让 data 的 property 能够响应数据变化，因此，给 vm.msg 赋值，就等价于 vm.\$data.msg。

better：

通过阅读源码去寻找这些问题的答案。

export function initState(vm: Component) {
  vm._watchers = [];
  const opts = vm.$options;
  if (opts.props) initProps(vm, opts.props);
  if (opts.methods) initMethods(vm, opts.methods);
  if (opts.data) {

  } else {

  if (opts.watch && opts.watch !== nativeWatch) {

```javascript
function initData(vm: Component) {
  let data = vm.$options.data;
  data = vm._data = typeof data === 'function' ? getData(data, vm) : data || {};
  if (!isPlainObject(data)) {
    data = {};
    process.env.NODE_ENV !== 'production' &&
      warn(
        'data functions should return an object:\n' +
          'https://vuejs.org/v2/guide/components.html#data-Must-Be-a-Function',
        vm
      );
  }
  // proxy data on instance
  const keys = Object.keys(data);
  const props = vm.$options.props;
  const methods = vm.$options.methods;
  let i = keys.length;
  while (i--) {
    const key = keys[i];
    if (process.env.NODE_ENV !== 'production') {
      if (methods && hasOwn(methods, key)) {
        warn(
          `Method "${key}" has already been defined as a data property.`,
          vm
        );
      }
    }
    if (props && hasOwn(props, key)) {
      process.env.NODE_ENV !== 'production' &&
        warn(
          `The data property "${key}" is already declared as a prop. ` +
            `Use prop default value instead.`,
          vm
        );
    } else if (!isReserved(key)) {
      proxy(vm, `_data`, key);
    }
  }
  // observe data
  observe(data, true /* asRootData */);
}
```

- proxy

```javascript
const sharedPropertyDefinition = {
  enumerable: true,
  configurable: true,
  get: noop,
  set: noop,
};

// proxy(vm, `_data`, key)
export function proxy(target: Object, sourceKey: string, key: string) {
  sharedPropertyDefinition.get = function proxyGetter() {
    return this[sourceKey][key];
  };
  sharedPropertyDefinition.set = function proxySetter(val) {
    this[sourceKey][key] = val;
  };
  Object.defineProperty(target, key, sharedPropertyDefinition);
}
```

回到刚才的例子：

```vue
<template>
  <div>{{ msg }}</div>
</template>

<script>
export default {
  name: 'home',
  data() {
    return {
      msg: '123',
    };
  },
  created() {
    console.log('msg', this.msg); // 123
    console.log('_data', this._data.msg); // 123
    console.log('$option', this.$options.data().msg); // 123

    this._data.msg = '456';
    console.log('after:msg', this.msg); // 456
    console.log('after:_data.msg', this._data.msg); // 456
    console.log('after:$option.msg', this.$options.data().msg); // 123
  },
};
</script>
```

给`this.msg`赋值，其实就是通过代理给`this._data.msg`赋值，`this._data.msg`改变了，因为`this.msg`是通过代理访问的`this._data.msg`，自然而然的就跟着变了。

到这里，就有人要问了，等等，讨论了这么久，我还有两个问题不明白：

1. 为什么`this.msg`就能访问到`this._data.msg`的值呢。

2. 为什么`this.$options.data`又是怎么拿到`msg`的值的，又是为什么给`this._data.msg`赋值，他不会变呢？

别急，我们通过实例来说明。

- 问题 1

```javascript
let vm = {
  _data: {
    msg: 123,
  },
};

const sharedPropertyDefinition = {
  enumerable: true,
  configurable: true,
  get() {},
  set() {},
};

function proxy(target, sourceKey, key) {
  sharedPropertyDefinition.get = function proxyGetter() {
    return vm[sourceKey][key]; // 做了个小变化，这里把原来的this改为了vm
  };
  sharedPropertyDefinition.set = function proxySetter(val) {
    vm[sourceKey][key] = val; // 做了个小变化，这里把原来的this改为了vm
  };
  Object.defineProperty(target, key, sharedPropertyDefinition);
}

proxy(vm, `_data`, 'msg');

vm; // {msg: 123, _data: {msg:123}}

vm.msg = 456;

vm; // {msg: 456, _data: {msg:456}}
```

`proxy`函数里边的`this`其实就是`vm`实例，可以回到刚才`initMixin`里查看。

```javascript
function initMixin(Vue: Class<Component>) {
  Vue.prototype._init = function (options?: Object) {
    const vm: Component = this;
    // ...
  };
}
```

因此，这就是为什么`this.msg`能获取到我们在`data`的 `msg`定义的值，并且通过给`this.msg`赋值，`data`里边的`msg`也会更着变了。
